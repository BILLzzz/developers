Basics

The feed is hosted at Amazon in Ireland and there is no limt on how often you download it. But generally we recommend you to cache the result in an sutiable way for you so that the user does not have to wait for the server to download the feed. In all the samples we use caching in some kind and as far as possible we describe other caching solution as well. How long you do want to cache is completely op to you, but keep in mind that the reviews, trustscore evt. does only update as often as you do. So caching to long will result in the data not being updated often enough, and to often result in bader performance for the user. An sample feed can be download from here http://s.trustpilot.com/tpelements/917278/f.json.gz. This feed is for the demo webshop www.demoshop.com. You can see the reviews on this domain here Demoshop and you are also welcome to create more reviews on this site as it is for demo usage only.

JSON Documentation

We deliver the feed as a single JSON object.

File location

The link for your specific JSON file can be found at http://b2b.trustpilot.com/Modules/Plugins (login required).
A demo feed can be found at http://s.trustpilot.com/tpelements/917278/f.json.gz.

Compression

The JSON feed file is zipped with gzip.

Elements of the feed

The feed is a tree structure of elements. Each attribute can either have a simple type like int or string, or a complex type. The type is described in brackets.

Feed root

  • Categories [Array of Category]
    The categories of this domain. May contain zero to multiple elements.
  • DomainName [string]
    The name of the domain, e.g. "demoshop.com".
  • FeedUpdateTime [Time]
    The time this feed was generated.
  • ReviewCount [ReviewCount]
    Total count of reviews and distribution of revers per rating.
  • ReviewPageUrl [string]
    The url for the review page of this domain on trustpilot. Use this to link to all the reviews.
  • Reviews [Array of Review]
    A list of up to 10 reviews of this domain.
    These reviews are selected by:
    • The reviews that the administrator of the domain have selected for the feed
    • Select from the remaining reviews, order by rating descending, then by date descending
  • TrustScore [TrustScore]
    Information about the score of the domain, measured in both rating (0-100) and number of stars (1-5).

Category

  • Count [int]
    The amount of domains in the category.
  • ImageUrls [Hash of string, string]
    A dictionary of urls to images showing the position of the domain in the category, e.g. in the top 3.
    The image is in different sizes and can be found with the keys, "i100", "i120", "i140", "i180", "i220", "i280" for the six different pixel sizes.
  • Name [string]
    The name of the category localized to the language of the domain.
  • Position [int]
    The position in the category.

Time

  • Human [string]
    A human readable representation of the time localized to the language of the domain. e.g. '4 October 2011 10:16:52 GMT'
  • HumanDate [string]
    A human readable representation of the date part of the time localized to the language of the domain. E.g. '15. Oct'
  • UnixTime [long]
    The number of seconds since January 1st, 1970 to this time.

ReviewCount

  • DistributionOverStars [Array of int]
    The number of reviews grouped by number of stars. E.g. [0, 1, 5, 4, 12] for a domain that have zero 1-starred reviews, 1 2-starred review, 5 3-starred reviews, 4 4-starred reviews and 12 5-starred reviews.
  • Total [int]
    The total amount of reviews on this domain.

Review

  • CompanyReply [string]
    The company reply of this review if any, otherwise null
  • Content [string]
    The main content of the review.
  • Created [Time]
    The creation time of the review.
  • Title [string]
    The title of the review.
  • TrustScore [TrustScore]
    The score of the review.
  • User [User]
    The author of the review.

TrustScore

  • Human [string]
    A description of the score, localized to the language of the domain. E.g. 'Excellent' or 'Good'.
  • Score [int]
    A score from 0 to 100.
  • Stars [int]
    A number of start from 1 to 5.
  • StarsImageUrls [Hash of string, string]
    Urls for images that shows the number of stars. The images come in three different sizes which can be found on the keys "small", "medium", and "large".

User

  • City [string]
    The city of the user if any, otherwise null.
  • HasImage [bool]
    True if the user have a profile image, otherwise false.
  • ImageUrls [Hash of string, string]
    Urls for the profile image in different sizes. If the user does not have a profile picture, the url will point to a default profile image.
    The sizes can be found on these keys: "i24", "i35", "i64", "i73".
  • IsVerified [bool]
    True if there is submitted an order id along with the review, otherwise false.
  • Locale [string]
    The locale or culture of the user. E.g. "en-GB" for British.
  • Name [string]
    The name of the user.
  • ReviewCount [int]
    The total number of reviews written by this user.

Search engine optimization

If you want to show your Trustpilot reviews on our webpage the widget is a great and easy way of showing them. But the widget is an iFrame solution and therefor you will not be any SEO juice from all the nice reviews. For that reason Trustpilot also provide a JSON feed with the same information as the widget just in JSON format. This feed can be used to build the same widget and just for the propose of getting SEO for the reviews or it can be used for building a widget that support you design the way you thing it will best support your business.

There has been build some samples in different languages. If your favorite language is not ammong them you are welcome to push i to the source and we will add it with the other samples with all credits to you.

C# .NET Sample

Here is an sample of how to download the JSON feed and parsing it to an set of models in C# .NET.

The sample uses System.Web.Caching for caching the result as this is highly recommende to use some kind of cache. Caching in this situation makes sure that the user does not have to wait for the server to download the feed and parse it every time.

There is several other way of doing this caching, one way could be just to download the feed and save it to local disk, and then every ones in a will download the file again. You could also parse this to you database and then get the data from there at every request. In all cases it could be an good idea to have an service running in the background that does fetch and parse (if need) the feed. This has the advantage that the user never has to wait for the data to download and parse. This is how ever not necessary as the feed normally is not bigger then 3-4KB.

If the FeedModel is not in the cache the code downloads the feed directly from the server. It then deserializes the string downloaded to an the FeedModel created for the purpose. The FeedModel can then be used to show the data in the frontend.
